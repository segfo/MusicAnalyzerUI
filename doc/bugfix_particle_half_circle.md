# バグ修正: パーティクルが下半分にしか出ない問題

## 概要

`viz_canvas.rs` のパーティクル生成において、360度全方向に放出されるはずのパーティクルが
下半分（約0〜180度）にしか現れないという視覚的バグ。

## 原因

`AnimState::rand()` 関数の実装ミス。

```rust
// 修正前 (バグあり)
fn rand(&mut self) -> f64 {
    self.rng = self.rng.wrapping_mul(6364136223846793005).wrapping_add(1442695040888963407);
    (self.rng >> 33) as f64 / (u32::MAX as f64)
}
```

### 問題の詳細

| 項目 | 値 |
|------|-----|
| `self.rng` の型 | `u64` |
| `self.rng >> 33` の最大値 | `2^31 - 1 = 2,147,483,647` (31ビット) |
| 除数 `u32::MAX` | `2^32 - 1 = 4,294,967,295` |
| `rand()` の実際の返り値範囲 | `[0.0, 0.5)` ← **本来は [0.0, 1.0) であるべき** |

`rand()` が最大 0.5 しか返さないため、`rand_range(0.0, 2π)` が `[0, π)` の範囲しか
生成できなかった。Canvas座標系では sin(0〜π) は常に正（下方向）なので、
パーティクルが画面の下半分にしか現れなかった。

## 修正内容

除数を `u32::MAX` から `(u64::MAX >> 33)` に変更し、
`rand()` の返り値が正しく `[0.0, 1.0)` になるよう修正。

```rust
// 修正後
fn rand(&mut self) -> f64 {
    self.rng = self.rng.wrapping_mul(6364136223846793005).wrapping_add(1442695040888963407);
    (self.rng >> 33) as f64 / ((u64::MAX >> 33) as f64)
}
```

`u64::MAX >> 33` は `2^31 - 1` と等しく、分子と分母のビット幅が一致するため
`[0.0, 1.0)` の範囲が正しく得られる。

## 影響範囲

`rand()` を呼ぶすべての `rand_range()` 呼び出しが影響を受けていた。

- パーティクル生成角度 → **これが主症状**
- スポーン半径のばらつき
- 速度のばらつき
- サイズのばらつき
- 色相オフセット
- decay / friction のばらつき

上記すべてが `[lo, lo + (hi-lo)/2)` の半分の範囲しかカバーできていなかった。
